## 七、如果你想自己继续往数据库里加数据，应该怎么做

## 方式 A：通过接口添加

适合日常补数据，尤其是你想走真实后端流程时。

建议顺序：

1. 先用管理员或挂号员账号新增科室
2. 再新增医生
3. 再新增排班
4. 再补患者或让患者自行注册
5. 再补药品

当前主要接口：

- `POST /api/departments`
- `POST /api/doctors`
- `POST /api/schedules`
- `POST /api/patients`
- `POST /api/drugs`
- `POST /api/system/roles`
- `PUT /api/system/users/{userId}/roles`

说明：

- 现在医生和患者通过后端建档后，账号与默认角色会自动补齐
- 医生建档后的默认密码是 `123456`
- 患者通过 `POST /api/patients` 建档后的默认密码也是 `123456`
- 患者通过 `POST /api/auth/register` 自注册时，密码以注册请求为准

## 方式 B：直接写 SQL

适合一次性批量准备演示环境。

建议规则：

- 科室编码用唯一值，例如 `DEMO-NEURO-02`
- 医生编号用唯一值，例如 `DOC-DEMO-10`
- 患者编号用唯一值，例如 `PAT-DEMO-10`
- 手机号必须唯一
- 账号名 `sys_user.username` 必须唯一
- 角色编码尽量复用现有标准编码，不要重复造 `ADMIN2` 这种角色

## 八、一个很重要的注意点：快速挂号还依赖 Redis 号源库存

如果你想演示“患者快速挂号”接口：

- `POST /api/registrations/quick`

除了数据库里的 `doctor_schedule` 数据之外，还需要初始化 Redis 中的号源库存键：

- `schedule:stock:{scheduleId}`

例如演示脚本里的排班 ID：

- `9401`
- `9402`
- `9403`
- `9404`

你可以执行类似命令：

```bash
redis-cli -p 6380 SET schedule:stock:9401 18
redis-cli -p 6380 SET schedule:stock:9402 20
redis-cli -p 6380 SET schedule:stock:9403 15
redis-cli -p 6380 SET schedule:stock:9404 20
```

说明：

- 数值建议与 `doctor_schedule.available_count` 保持一致
- 不初始化这些 Redis key，快速挂号流程可能无法正常演示
