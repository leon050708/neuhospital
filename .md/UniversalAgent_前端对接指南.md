# Universal Agent (全能智能体) 前端对接与交互指南

你好，前端同学！
后端目前已经完成了一次重大的架构升级，我们把“智能问诊”、“病历提取”、“导诊推荐”和“极速挂号”四大核心功能，全部揉进了一个无敌的 **Universal Agent (全能大模型)** 中。

这意味着你**不需要**去对接多个复杂的后端接口，整个问诊挂号流程，只需要一个朴实无华的聊天框即可搞定！

---

## 🚀 极简对接核心思路

整个应用只依赖两个基础接口：
1. **创建房间**：`POST /api/ai/chat/sessions` （进入页面时调用一次，拿到 `sessionId`）
2. **发送消息**：`POST /api/ai/chat/sessions/{sessionId}/messages` （患者每次点发送时调用）

**无论患者是想闲聊、看病，还是想挂号，你统统只需要调用第 2 个接口，把文本发给后端即可！所有的业务判断、去数据库查病历、扣除号源的脏活累活，全由大模型在后端自动完成。**

---

## 🎨 重点：号源选项卡的渲染交互

虽然前端只需要调一个接口，但为了患者的体验，我们需要在聊天气泡中做一点小小的“魔法解析”。

### 1. 识别并渲染挂号卡片
当患者在聊天中触发了挂号意图（比如患者说：“行，帮我挂个明天的号吧”），大模型会在回复文本的末尾，**附带一段固定的 JSON 字符串**。

后端的返回示例 (`response.data`)：
```text
好的，我查了一下，心血管内科明天有以下医生出诊，您可以看下时间安排，选择一位医生进行预约哦。
```json
{
  "type": "SCHEDULE_OPTIONS",
  "options": [
    {"scheduleId": 101, "doctorName": "张主任", "timeSlot": "上午", "remainQuota": 5},
    {"scheduleId": 102, "doctorName": "李医生", "timeSlot": "下午", "remainQuota": 10}
  ]
}
```

**前端要做的事：**
1. 收到大模型的回复文本后，使用正则表达式或者字符串切割，把 ````json` 和 ````` 中间的那段 JSON 抠出来。
2. 上半部分的纯文本正常渲染成 AI 的文字气泡。
3. 把抠出来的 JSON 解析为对象。如果发现 `type === 'SCHEDULE_OPTIONS'`，就在气泡下方渲染出两个漂亮的按钮：【预约张主任(上午) - 余5】、【预约李医生(下午) - 余10】。

### 2. 用户点击卡片后的极简交互
当患者点击了前端渲染出的【预约张主任】按钮时，前端需要做什么？调挂号接口吗？**不需要！**

**前端要做的事：**
只需像患者打字一样，偷偷组装一句话，再次调用普通的发送消息接口 `/api/ai/chat/sessions/{sessionId}/messages`。
请求体示例：
```json
{
  "content": "我要预约排班ID为101的号"
}
```
*（提示：你可以把这句话以患者的名义渲染在聊天窗口右侧，也可以静默发送，推荐显示在右侧以增加真实感。）*

**后端大模型收到这句话后：**
它会秒懂患者点了哪个按钮，然后它会在后端**自动调用**系统的底层挂号工具，完成防超卖的真实扣号！
随后，大模型会回复你：`"挂号成功！您的单号是 XXXXX。"`，你直接把它显示在聊天框里就大功告成了！
