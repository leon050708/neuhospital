# 微服务网关第一阶段启动说明

当前阶段先保留原单体后端，只新增 Spring Cloud Gateway 和 Nacos 服务发现：

- `backend-service`：原 `neuhospital` 后端，默认端口 `8081`
- `gateway-service`：新增网关服务，默认端口 `10010`
- `file-service`：独立文件服务，默认端口 `10020`
- `patient-service`：独立患者服务，默认端口 `10021`
- `doctor-service`：独立医生科室服务，默认端口 `10022`
- Nacos 控制台：`http://localhost:8080`
- Nacos 注册发现地址：`127.0.0.1:8848`
- Nacos gRPC 端口：`9848`

注意：浏览器打开 Nacos 控制台用 `8080`，但 Java 客户端注册服务要连 `8848`。不要把 `NACOS_SERVER_ADDR` 写成 `127.0.0.1:8080`，否则客户端会尝试连接 `9080` 并启动失败。

## 1. 启动顺序

推荐固定按这个顺序启动：

1. 启动 Nacos。
2. 启动 Redis、Kafka、MinIO。
3. 启动后端 `backend-service`。
4. 启动网关 `gateway-service`。
5. 通过网关 `10010` 访问接口。

## 2. 检查 Nacos 端口

确认 Nacos 已启动，并且至少监听这些端口：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:8848 -sTCP:LISTEN
lsof -nP -iTCP:9848 -sTCP:LISTEN
```

正确含义：

- `8080`：Nacos 控制台页面。
- `8848`：Nacos OpenAPI 和服务注册发现主端口。
- `9848`：Nacos 2.x/3.x 客户端 gRPC 端口，通常是 `8848 + 1000`。

## 3. 启动基础设施

在项目根目录启动 Redis、Kafka、MinIO：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1
docker compose -f infra/compose.yml up -d
```

检查端口：

```bash

lsof -nP -iTCP:9094 -sTCP:LISTEN
lsof -nP -iTCP:9005 -sTCP:LISTEN
```

## 4. 启动后端服务

后端必须在项目根目录启动：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1
./mvnw spring-boot:run
```

后端默认配置：

- `spring.application.name=backend-service`
- `server.port=8081`
- `spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848`

看到以下日志表示启动成功：

```text
Tomcat started on port 8081
Success to connect to server [127.0.0.1:8848]
nacos registry, DEFAULT_GROUP backend-service ...:8081 register finished
Started NeuhospitalApplication
```

检查后端端口：

```bash
lsof -nP -iTCP:8081 -sTCP:LISTEN
```

## 5. 启动网关服务

网关必须在 `gateway-service` 目录启动：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1/gateway-service
```

网关默认配置：

- `spring.application.name=gateway-service`
- `server.port=10010`
- `spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848`

看到以下日志表示启动成功：

检查网关健康状态：

## 7. 访问方式

前端或接口工具以后优先访问网关，不直接访问后端：

- 登录：`POST http://localhost:10010/api/auth/login`
- 当前用户：`GET http://localhost:10010/api/auth/me`
- Swagger：`http://localhost:10010/swagger-ui.html`
- Knife4j：`http://localhost:10010/doc.html`
- 网关健康检查：`http://localhost:10010/actuator/health`

网关会把这些路径转发到 Nacos 中的 `backend-service`：

- `/api/**`
- `/v3/api-docs/**`
- `/swagger-ui/**`
- `/swagger-ui.html`
- `/doc.html`

当前网关已经按真实控制器前缀拆成领域路由。已拆出的文件、患者、医生科室、挂号领域都支持通过环境变量切到独立服务；未设置变量时仍回退到 `backend-service`。

| 网关路由分组 | 当前路径 | 当前目标服务 | 后续可替换为 |
| --- | --- | --- | --- |
| 认证与系统 | `/api/auth/**`, `/api/system/**` | `lb://backend-service` | `lb://auth-service` |
| 患者 | `/api/patients/**` | `${PATIENT_SERVICE_URI:lb://backend-service}` | `lb://patient-service` |
| 医生科室 | `/api/doctors/**`, `/api/departments/**` | `${DOCTOR_SERVICE_URI:lb://backend-service}` | `lb://doctor-service` |
| 排班 | `/api/schedules/**` | `${REGISTRATION_SERVICE_URI:lb://backend-service}` | 已支持切到 `registration-service` |
| 挂号队列 | `/api/registrations/**`, `/api/queue/**` | `${REGISTRATION_SERVICE_URI:lb://backend-service}` | 已支持切到 `registration-service` |
| 门诊 | `/api/outpatient/**` | `lb://backend-service` | `lb://outpatient-service` |
| 检查检验处置 CT | `/api/check-requests/**`, `/api/check-results/**`, `/api/inspection-requests/**`, `/api/inspection-results/**`, `/api/disposal-requests/**`, `/api/ct-analysis/**` | `lb://backend-service` | `lb://inspection-service` |
| 药房处方 | `/api/drugs/**`, `/api/prescriptions/**`, `/api/pharmacy/**` | `lb://backend-service` | `lb://pharmacy-service` |
| 支付 | `/api/payment/**` | `lb://backend-service` | `lb://payment-service` |
| 文件 | `/api/files/**` | `${FILE_SERVICE_URI:lb://backend-service}` | `lb://file-service` |
| AI | `/api/ai/**` | `lb://backend-service` | `lb://ai-service` |

## 8. 一键检查网关

后端和网关都启动后，可以在项目根目录执行：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1
./scripts/check-gateway.sh
```

这个脚本会检查：

- 网关健康状态。
- Nacos 服务列表。
- `/v3/api-docs` 是否能经网关访问。
- `/api/auth/me` 是否被网关鉴权拦截。

如果已经启动独立 `file-service`，可以在启动网关时切换文件路由：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1/gateway-service
FILE_SERVICE_URI=lb://file-service ../mvnw spring-boot:run
```

不设置 `FILE_SERVICE_URI` 时，`/api/files/**` 仍然转发到 `backend-service`，不会影响当前后端。

如果已经启动独立 `patient-service`，可以在启动网关时切换患者路由：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1/gateway-service
PATIENT_SERVICE_URI=lb://patient-service ../mvnw spring-boot:run
```

如果文件服务和患者服务都要切到独立服务：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1/gateway-service
FILE_SERVICE_URI=lb://file-service PATIENT_SERVICE_URI=lb://patient-service ../mvnw spring-boot:run
```

如果医生科室服务也要切到独立 `doctor-service`：

```bash
cd /Users/aijiaqi/shixun_2026summer/hou1/gateway-service
FILE_SERVICE_URI=lb://file-service PATIENT_SERVICE_URI=lb://patient-service DOCTOR_SERVICE_URI=lb://doctor-service ../mvnw spring-boot:run
```

如果网关端口或 Nacos 地址改过，可以覆盖变量：

```bash
GATEWAY_URL=http://127.0.0.1:10011 NACOS_URL=http://127.0.0.1:8848 ./scripts/check-gateway.sh
```

## 9. 安全策略

网关会放行：

- `/api/auth/login`
- `/api/auth/register`
- `/api/auth/refresh`
- `/api/auth/logout`
- `/v3/api-docs/**`
- `/swagger-ui/**`
- `/swagger-ui.html`
- `/doc.html`
- `/webjars/**`
- `/actuator/**`

其他接口必须带：

```text
Authorization: Bearer <accessToken>
```

网关只做 JWT 签名、过期时间、issuer 的前置校验。原后端的 Spring Security 和 Redis 登录态校验仍然保留，避免绕过网关时失去安全保护。

## 10. 当前网关限流

网关已对两个入口加 Redis 限流：

- `/api/auth/login`：按 IP 限流，默认每秒 5 个令牌，突发容量 10
- `/api/registrations/quick`：优先按用户 ID 限流，没有用户 ID 时按 IP 限流，默认每秒 10 个令牌，突发容量 20

挂号服务内部原有的 Redis Lua 限流和 Redisson 防重复提交仍然保留，网关限流只是第一层保护。

### 10.1 当前已拆服务的网关切流变量

```text

DOCTOR_SERVICE_URI=lb://doctor-service
REGISTRATION_SERVICE_URI=lb://registration-service
```

完整启动示例：

## 11. 常见问题

### 11.1 Nacos 控制台能打开，但服务启动失败

如果日志里出现：

```text
Server check fail, please check server 127.0.0.1, port 9080 is available
Client not connected, current status:STARTING
```

说明 `NACOS_SERVER_ADDR` 写错了。你把注册发现地址写成了 `127.0.0.1:8080`。

正确值是：

```text
127.0.0.1:8848
```

原因：

- `8080` 是控制台端口。
- Nacos 客户端会用主端口加 `1000` 连接 gRPC。
- 如果主端口写成 `8080`，客户端就会连 `9080`。
- 你的 Nacos 实际监听的是 `8848` 和 `9848`。

### 11.2 日志里出现 BeanPostProcessorChecker 警告

类似日志：

```text
Bean ... is not eligible for getting processed by all BeanPostProcessors
```

这是 Spring Cloud LoadBalancer 的启动期警告，不是启动失败原因。只要后面出现 `Started ...Application` 就可以忽略。

### 11.3 日志里出现 Netty DNS macOS 警告

类似日志：

```text
Unable to load io.netty.resolver.dns.macos.MacOSDnsServerAddressStreamProvider
```

这是 macOS 下 Netty DNS 原生库缺失警告。当前本地开发可以先忽略；如果后续出现实际 DNS 解析问题，再补 `netty-resolver-dns-native-macos` 依赖。

### 11.4 端口被占用

检查占用：

```bash
lsof -nP -iTCP:8081 -sTCP:LISTEN
lsof -nP -iTCP:10010 -sTCP:LISTEN
```

如果 `8081` 被占用，可以临时换后端端口：

```bash
BACKEND_SERVER_PORT=8082 ./mvnw spring-boot:run
```

如果 `10010` 被占用，可以临时换网关端口：

```bash
GATEWAY_SERVER_PORT=10011 ../mvnw spring-boot:run
```

### 11.5 启动目录错误

后端启动目录：

```bash
/Users/aijiaqi/shixun_2026summer/hou1
```

网关启动目录：

```bash
/Users/aijiaqi/shixun_2026summer/hou1/gateway-service
```

如果在根目录启动的是后端；如果在 `gateway-service` 目录启动的是网关。
